将 Hexo 部署到 GitHub Pages

2021年8月06日
阅读时长: 1 分钟

1.登录Github创建一个名为 “用户名 .github.io” 的新公共存储库repository，其中用户名是你在 GitHub 上的用户名（或组织名称）。如果存储库的第一部分与你的用户名不完全匹配，它将无法工作，因此请确保正确。

2.安装node.js,Git。执行 node -v,git --version 指令检查是否安装成功，安装成功会输出版本号。生成SSH密钥ssh-keygen -t rsa -C "your_emial@examle.com"添加到 “Settings” 下的 “Deploy keys”。

# 设置user.name和user.email
git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub注册邮箱"

3.部署Hexo 。执行 npm install hexo-deployer-git --save 安装hexo-deployer-git插件，修改_config.yml文件内deploy的配置，执行 hexo deploy 部署到Github Pages。

# 安装hexo
npm install hexo-cli -g
hexo init blog
cd blog
npm install
# _config.yml
type: git
  repo: git@github.com:<username>/<username>.github.io.git
  branch: master

4.自定义域（可选）。打开dos面板执行 ping 用户名.github.io 获取公网ip，登录域名提供平台，修改DNS添加A记录，将记录值更改为公网ip。打开Github 用户名.github.io的存储库单击 “Settings” 面板下的 “Pages”，添加自定义域并保存。为避免每次部署都要重新添加自定义域，可在hexo文件夹中的 “source” 内添加一个名为 “CNAME” 的文件并在其内写入域名。
